  <action>
Replace the existing "Edit Interface" modal (the one opened by `openEdit()`) with a new port configuration panel. This panel should open when a port is clicked in either renderDeviceSwitch or renderGenericSwitch.

**Port Config Panel design (use the existing Modal component):**

Title: "Configure: {port.name}" (e.g., "Configure: eth0")

Section 1 — Port Info (read-only, top of modal):
- Show port name, MAC, port type, speed, link status in a compact info bar
- Use the existing zone color styling for the current PVID zone

Section 2 — Primary Zone (PVID):
- Label: "Primary Zone (PVID)"
- Render as a list of radio-button-style options, one per zone from the zones array (fetched by Plan 01).
- Each option shows: zone color dot + zone name (uppercase) + VLAN ID in parentheses.
- Example: [green dot] LAN (VLAN 10), [red dot] WAN (VLAN 0), [amber dot] DMZ (VLAN 3001), [gray dot] VOID (VLAN 1)
- The currently active PVID's zone is pre-selected.
- Selecting a different zone updates the local form state (does NOT save immediately).
- Special case: PVID 0 = WAN. Show "WAN (no internal VLAN)" with red styling.
- Special case: PVID 1 = Void. Show "VOID (isolated — all traffic dropped)" with dark styling.

Section 3 — Tagged VLANs:
- Label: "Additional Tagged VLANs"
- Show a checklist (checkboxes) of all non-void zones (skip void zone from checklist).
- Each checkbox: zone color dot + zone name + VLAN ID.
- Pre-check the VLANs that are in the port's current `tagged_vlans` array.
- The PVID's own VLAN should NOT appear in this checklist (it's already the primary).
- If PVID is 0 (WAN) or 1 (void), disable the tagged VLANs section entirely (WAN ports don't carry internal VLANs; void ports don't carry anything).

Section 4 — Actions:
- "Save" button (primary) — calls `api.updatePort(portName, { pvid, tagged_vlans })`.
- "Cancel" button (secondary) — closes modal without saving.
- While saving, show a loading state on the Save button.
- On success: show toast "Port {name} updated", close modal, call `load()` to refresh the switch panel (no full page reload).
- On error: show toast with error message, keep modal open.

**State management:**
- Add state: `configPort: PortConfig | null` (the port being configured, null = panel closed)
- Add state: `configPvid: number` (local form state for PVID selection)
- Add state: `configTagged: number[]` (local form state for tagged VLAN checklist)
- Add state: `saving: boolean`

When a port is clicked:
1. Find the port in the interfaces array by name
2. If the interface has pvid and tagged_vlans data, use it directly to populate config state
3. If not (fallback), fetch via `api.getPort(name)` and populate
4. Set configPort, configPvid, configTagged, open modal

**Keep the old edit modal for bridges and VLAN sub-interfaces:**
The old `openEdit()` modal should still work for `br-*` interfaces and VLAN sub-interfaces (they don't have PVID config). Only physical ports (no dot in name, not starting with br-) should open the new config panel.

**Wiring into switch panel:**
In renderPort() (both device-specific and generic), change the `onClick` to call the new config panel opener instead of `openEdit()`.
