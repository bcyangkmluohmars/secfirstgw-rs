**1. Update `InterfaceInfo` struct:**

```rust

**2. Update `configure()` function:**
The INSERT statement currently writes `role`:
```sql
INSERT INTO interfaces (name, mac, ips, mtu, is_up, role)
VALUES (?1, ?2, ?3, ?4, ?5, ?6)
ON CONFLICT(name) DO UPDATE SET ...
```
Change to:
```sql
INSERT INTO interfaces (name, mac, ips, mtu, is_up, pvid, tagged_vlans)
VALUES (?1, ?2, ?3, ?4, ?5, ?6, ?7)
ON CONFLICT(name) DO UPDATE SET
    mac = excluded.mac,
    ips = excluded.ips,
    mtu = excluded.mtu,
    is_up = excluded.is_up
```
PVID is only set on first discovery (same principle as the old role — never overwrite user config on reboot). The params change: replace `iface.role` with `iface.pvid` as i32 and add `serde_json::to_string(&iface.tagged_vlans)` for tagged_vlans.

Update the tracing::info! to log pvid instead of role.

**3. Update `detect_interfaces_for_platform()`:**

- Docker mode: all interfaces → pvid=10 (LAN)
- BareMetal WAN interfaces → pvid=0 (WAN is separate world)
- BareMetal MGMT interface → pvid=3000
- BareMetal other → pvid=10 (LAN default)
- The `if iface.role == "wan"` check in Docker mode becomes `if iface.pvid == 0`
- tagged_vlans stays empty (default) everywhere during detection — trunk config is user-configured later

**4. Update WAN config auto-creation:**
The loop `for iface in &interfaces { if iface.role == "wan" ...` becomes `if iface.pvid == 0 ...` (pvid=0 means WAN port).

**5. Update `guess_role()` → `guess_pvid()`:**
Rename function to `guess_pvid(name: &str) -> u16`. Same logic but returns PVID numbers:
- "lo" → 0 (loopback, not a VLAN port)
- wg/tun/tap → 0 (VPN, not a VLAN port)
- docker/br-/veth → 0 (container, not a VLAN port)
- wifi/wlan → 10 (LAN)
- virbr → 0 (virtual, not a VLAN port)
- eth0/ens/enp (first physical) → 0 (WAN)
- everything else → 10 (LAN default)

Also add `guess_tagged_vlans()` that always returns `vec![]` — tagged VLANs are never auto-detected, always user-configured. This makes the intent explicit.

    continue;

```rust

if net.zone == "void" {

Since the caller knows the zone, pass zone info through.

Simplest correct approach: Change the function signature to `build_port_string(vlan_id: i32, zone: &str, sw: &SwitchLayout) -> String`. Then:
- `zone == "lan"` → LAN ports untagged, MGMT port excluded (same as current VLAN 1 behavior)

**2. Update `interface_update()` handler:**
Remove the role validation block (`valid_roles` check).
Remove `role` from the dynamic UPDATE builder.
Remove `vlan_id` from the dynamic UPDATE builder.

Add `pvid` to the dynamic UPDATE builder:
```rust
if let Some(pvid) = body.get("pvid").and_then(|v| v.as_i64()) {
    // VLAN ID range: 0 (WAN/special) or 1-4094
    if pvid != 0 && !(1..=4094).contains(&pvid) {
        return validation error "pvid must be 0 (WAN) or 1-4094";
    }
    sets.push("pvid = ?");
    params.push(Box::new(pvid));
}
```

Add `tagged_vlans` to the dynamic UPDATE builder:
```rust
if let Some(tagged) = body.get("tagged_vlans").and_then(|v| v.as_array()) {
    // Validate each VLAN ID is 1-4094
    for vlan in tagged {
        if let Some(v) = vlan.as_i64() {
            if !(1..=4094).contains(&v) {
                return validation error "tagged VLAN IDs must be 1-4094";
            }
        } else {
            return validation error "tagged_vlans must be array of integers";
        }
    }
    // Store as JSON string
    let json_str = serde_json::to_string(tagged).unwrap_or_else(|_| "[]".to_string());
    sets.push("tagged_vlans = ?");
    params.push(Box::new(json_str));
}
```
